How to Expand This Tool to Other Role Functions

This tool is currently built for Product Management. Expanding it to cover Engineering, Design, Data Science, or other functions requires updating four data objects in the code. Here is exactly what to change and how.

1 Add new domains to Market Intel

Find the marketData object in the JavaScript. Each key is a domain (e.g. "ai", "product"). To add Engineering, duplicate an existing entry and update the values:

"swe": { scarcity: "High", tenure: "2.4 yrs", donors: "Google, Amazon, Microsoft", trend: "AI eng roles heating up fast", alert: "Staff+ engineers at Google have 4yr RSU refresh — flag cliff dates", inflow: [ {company:"Google", pct:30, note:"Largest source. L5/L6 pipeline."}, {company:"Amazon", pct:22, note:"SDE II/III ready to move at 2yr cliff."}, ... ], topPaths: ["Google L5 → Meta E5", "Amazon SDE III → Meta E6"], watchout: "Bar for E6+ at Meta requires strong system design signal." }

Then add a new <option> to the Market Intel domain dropdown in the HTML.

2 Add role types to Sourcing

Find the sourcingData object. Each key is a role type (e.g. "pm", "ai-pm"). Add a new entry for each role you want to source:

"swe-backend": { clusters: ["L5/L6 engineers at Google, Amazon, Meta with distributed systems depth", ...], donors: ["Google (L5 pipeline)", "Amazon (SDE III)", "Stripe", ...], search: "LinkedIn: "Software Engineer" AND ("distributed" OR "backend") ...", watchouts: ["Verify system design ownership, not just contribution", ...] }

Then add a new <option> to the Role Type dropdown in the Sourcing tab HTML.

3 Update Level Calibration for new functions

Find the levelMap object. Each key is a company, and the value is an array of 7 title progressions. These are currently PM titles. To support Engineering, you have two options:

• Option A (simple): Add an "Engineering" toggle that swaps the levelMap titles to SWE equivalents (e.g. L3→L7 at Google, SDE I–III at Amazon).

• Option B (full build): Create a separate levelMapEng object and a function toggle to switch between PM and Engineering calibration mode. This is the cleanest long-term approach.

Also update metaLevels and metaDescs arrays to reflect Meta Engineering levels (E3–E9) instead of IC3–IC9.

4 Update Competitive landscape

Find the compData array. Each object has a roles array that controls which role filters show that company. Add your new role keys to the relevant companies:

{name:"Google", roles:["pm","ai-pm","platform","growth","pm-mgr","swe-backend","swe-ml"], ...}

Also add new teams values relevant to that function (e.g. for Google SWE: "Search Backend", "YouTube Infra", "Ads Systems").
